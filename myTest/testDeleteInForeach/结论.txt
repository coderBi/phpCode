①： array_splice只适合删除一次，不适合放在foreach里面。因为如果是$value会导致删不完全，如果是&$value，由于在foreach里面
操作的是原数组本身，而array_splice会导致原数组脚标的变动，通过测试foreach里面每执行一个array_splice会导致foreach重新
从 0 脚标进行遍历。
②：剔除掉上面又从 0 脚标开始遍历的重复判断之外，一般程序可能在不符合删除的之后还有一些其他的操作，那么由于再次从 0 脚标进行
遍历会导致有些操作进行了多次（而有的操作程序中进行多次会出错的，例如像元素末尾追加一个字符串）
③：②中的再次重 0 脚标开始遍历，原因也很容易理解，因为 array_splice之后数组的脚标会被重置位 0 1 2 ...。举个例子，之前脚标是
 0 100 现在删除 0，那么 100脚标的元素成了 0 脚标。
④：纯粹删除元素，还是用unset比较合适。
